# LinkedIn launch post draft

I built a small job-search tool for anyone trying to stay sane during an active search: **Opportunity Radar**.

It’s a simple Kanban dashboard for tracking every opportunity from first inbound message to screen, founder/hiring-manager conversations, offer, pass, or rejection.

Why I wanted this:
- Recruiter emails and LinkedIn messages get scattered fast.
- The actual JD/history is often buried in LinkedIn.
- Follow-ups go stale unless there’s a visible system.
- A spreadsheet is useful, but not very motivating to use every day.

What it does:
- Tracks opportunities by stage: New → Screen → Founder/HM → Offer → Pass/Rejected
- Tags LinkedIn-originated opportunities so you know where to look for context
- Highlights stale follow-ups and high-priority roles
- Runs as a lightweight static dashboard
- Keeps private job-search data local by default

I’m sharing it publicly in case it helps other job seekers build a better operating system for their search.

Repo: https://github.com/llm-apprentice/opportunity-radar
